**Theming** in termcn means choosing a **terminal theme** (tokens for colors, borders, and spacing) and mounting **`ThemeProvider`** only when an application needs to override the defaults. Ink theme-aware components work without a provider; their standalone hooks install into `hooks/`. Theme packages install into `lib/terminal-themes/`.

## ThemeProvider

Install the provider when you want to customize the active theme or capability contexts:

For Ink:

```bash
npx shadcn@latest add @termcn/ink/theme-provider
```

For OpenTUI:

```bash
npx shadcn@latest add @termcn/opentui/theme-provider
```

### Usage

```tsx
import { ThemeProvider } from "@/components/ui/theme-provider";
import { defaultTheme } from "@/lib/terminal-themes/default";
import { Spinner } from "@/registry/ui/spinner";

export function Example() {
  return (
    <ThemeProvider theme={defaultTheme}>
      <Spinner label="Generating registry items" />
    </ThemeProvider>
  );
}
```

Ink components receive `use-theme`, `use-unicode`, and `use-motion` transitively as `registry:hook` items when needed. Import the hooks directly from the configured shadcn hooks alias:

```tsx
import { useTheme, useThemeUpdater } from "@/hooks/use-theme";
import { useUnicode } from "@/hooks/use-unicode";
import { useMotion } from "@/hooks/use-motion";
```

Each hook has a usable default context, so these imports do not require a mounted provider. OpenTUI retains its existing provider contract.

### Custom themes

```tsx
import { createTheme } from "@/components/ui/theme-provider";

export const oceanTheme = createTheme({
  name: "ocean",
  colors: {
    primary: "#0ea5e9",
    accent: "#22d3ee",
    background: "#0f172a",
    foreground: "#e2e8f0",
  },
});
```
