title: Registry
description: Learn how to use the termcn registry with the shadcn CLI.
---

The **termcn** registry ships Ink and OpenTUI terminal UI components and themes in the [shadcn registry format](https://ui.shadcn.com/docs/registry). Register the namespace once in `components.json`, then pull items by name with the CLI.

## Setup

Add a **registries** entry for this site. See the [shadcn registry documentation](https://ui.shadcn.com/docs/registry) for all `components.json` fields and behaviors.

```json
{
  "registries": {
    "@termcn": "https://termcn.dev/r/{name}.json"
  }
}
```

You can replace `@termcn` with another namespace; it only needs to match what you pass to `shadcn add`.

## Usage

Items are namespaced by base: `ink/` for Ink and `opentui/` for OpenTUI.

Install an Ink component:

```bash
npx shadcn@latest add @termcn/ink/spinner
```

Install an OpenTUI component:

```bash
npx shadcn@latest add @termcn/opentui/spinner
```

Install a theme for the matching base:

```bash
npx shadcn@latest add @termcn/ink/theme-dracula
```

```bash
npx shadcn@latest add @termcn/opentui/theme-dracula
```

Install the theme provider for each base:

```bash
npx shadcn@latest add @termcn/ink/theme-provider
```

```bash
npx shadcn@latest add @termcn/opentui/theme-provider
```

Without a registry alias, use the full URL (as on each docs page):

```bash
npx shadcn@latest add https://termcn.dev/r/ink/spinner.json
```

```bash
npx shadcn@latest add https://termcn.dev/r/opentui/spinner.json
```
